### Основные команды для работы с папками и файлами


- touch 'filename' - создать файл
- mkdir 'foldername' - создать папку
- pwd - узнать по какому адресу мы находимся (путь в проводнике)
- cd 'foldername' - перемещение в дочернюю папку относительно той, где находимся
- cd .. - перемещение в родительскую папку относительно той, где находимся
- rm 'filename' - удаление файла (или папки, если папка пустая)
- rm -r 'foldername' - рекурсивное удаление папки (со всеми вложенными в нее файлами и папками)

### Основные команды для работы с репозиториями


- git init - инициализация нового репозитория Git в текущей папке
- git status - отображение состояния текущей папки
- git add 'filename' - добавление файла к коммиту
- git commit -m 'Message' - выполнение коммита с комментарием
- git log - просмотр истории коммитов
- git push - загрузка всех изменений в ветку удаленного репозитория
- git clone 'link from github or other hub' - клонирование репозитория на локальный ПК
- git remote add origin 'link' - синхронизация локального и удалённого репозиториев
- git remote -v - проверка, что репозитории связались
- git push -u origin main - в ПЕРВЫЙ раз загрузить все коммиты из локального репозитория в удалённый с названием origin.
- git branch '%BRANCH_NAME' - создание новой ветки
- git checkout 'BRANCH_NAME' - переход в ветку

### Хеш, лог и HEAD


- Хеш коммита — это уникальный буквенно-цифровой идентификатор, который Git присваивает каждому сохранению (коммиту)
в истории проекта
- Лог коммита (git log) — это журнал истории изменений проекта в Git, отображающий упорядоченный список всех
фиксаций (коммитов). Он показывает уникальный хэш, автора, дату и сообщение для каждого снимка состояния, позволяя
отслеживать, кто, когда и что изменил. 
- HEAD в Git — это указатель (ссылка) на самый последний коммит (срез состояния) в текущей активной ветке.
Он определяет, какая версия кода сейчас находится в рабочей директории и станет «родителем» для следующего коммита.
По сути, HEAD указывает, где вы находитесь в истории разработки.
